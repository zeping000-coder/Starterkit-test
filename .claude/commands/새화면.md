## 입력 형식

`$ARGUMENTS`는 "영문슬러그 한글이름" 두 단어로 받습니다.
예: `/새화면 watchlist 관심종목`

- 영문슬러그: 폴더/파일 이름에 쓸 소문자 영문 (예: `watchlist`)
- 한글이름: 네비게이션과 화면 제목에 쓸 한글 (예: `관심종목`)

두 값이 모두 주어지지 않았다면, 사용자에게 되물어보고 진행하세요.

## 작업 순서

이 저장소는 `web/` 안에 Next.js 앱이 있고, 화면 3개(`/`, `/account`, `/candidates`)가 전부 동일한 3단계 패턴을 따릅니다. 아래 기존 파일들을 반드시 먼저 읽고, **같은 스타일(주석 톤, 타입 이름 규칙, 더미데이터 생성 방식)**로 새 화면을 만드세요.

1. **패턴 참고용으로 먼저 읽을 파일**
   - `web/src/lib/candidates.ts` (lib 파일 예시 — 타입 정의 + `getDummy<Domain>Snapshot()` 함수)
   - `web/src/app/api/candidates/route.ts` (얇은 GET Route Handler 예시)
   - `web/src/app/candidates/page.tsx` (TanStack Query + shadcn/ui 사용하는 클라이언트 컴포넌트 예시)
   - `web/src/components/layout/top-nav.tsx` (네비게이션 메뉴)

2. **`web/src/lib/<영문슬러그>.ts` 생성**
   - 상단에 한글 주석으로 "이 화면이 무엇을 보여주는지 + 나중에 실제 API/DB로 교체할 지점"을 설명 (기존 파일들 톤 참고)
   - 응답 타입(`<Domain>Response` 등)과 `getDummy<Domain>Snapshot()` 함수 작성
   - 시계열/차트 데이터가 필요하면 `src/lib/dummy-data.ts`의 `generateWalk` / `generateBiasedWalk`를 재사용 (새로 만들지 말 것)

3. **`web/src/app/api/<영문슬러그>/route.ts` 생성**
   - `getDummy<Domain>Snapshot()`을 호출해서 `NextResponse.json()`으로 반환하는 얇은 `GET` 함수만 작성 (비즈니스 로직 넣지 않기)

4. **`web/src/app/<영문슬러그>/page.tsx` 생성**
   - `"use client"` 클라이언트 컴포넌트
   - `useQuery({ queryKey: ["<영문슬러그>"], queryFn: ... })`로 `/api/<영문슬러그>` 호출
   - 로딩 상태는 `Skeleton`, 에러 상태는 `Card` + `text-destructive`, 정상 상태는 실제 데이터로 렌더링 (candidates/page.tsx와 동일한 구조)
   - 상단에 `<h1>`으로 한글이름 제목 + 부제로 "(현재는 더미 데이터입니다)" 문구 포함
   - 하단에 "마지막 갱신: {new Date(data.updatedAt).toLocaleString("ko-KR")}" 표시

5. **`web/src/components/layout/top-nav.tsx`의 `NAV_ITEMS`에 새 메뉴 추가**
   - `{ href: "/<영문슬러그>", label: "<한글이름>" }`를 배열에 추가

6. **주의사항**
   - Next.js 16 / recharts v3 / lightweight-charts v5 등 API가 필요하면 추측하지 말고 `web/node_modules/next/dist/docs/` 또는 Context7로 먼저 확인
   - Prisma 스키마(`prisma/schema.prisma`)는 사용자가 명시적으로 요청하지 않는 한 건드리지 말 것 (이 커맨드는 더미 데이터 화면 스캐폴딩까지만)
   - 작업 완료 후 `npx tsc --noEmit`으로 타입 에러 없는지 확인
